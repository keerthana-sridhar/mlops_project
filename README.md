End-to-end MLOps pipeline for malaria cell classification with:

- FastAPI backend (inference + APIs)
- Streamlit frontend (user interface)
- DVC (reproducible pipeline)
- MLflow (experiment tracking + model registry)
- Airflow (retraining pipeline)
- Prometheus + Grafana + Alertmanager (monitoring)

---

## 🚀 Quick Start (Evaluator Guide)

### 1. Clone the repository

```bash
git clone https://github.com/keerthana-sridhar/mlops_project.git
cd mlops_project
```

### 2. Setup environment

```bash
cp .env.example .env
```

(Optional) Generate Airflow Fernet key:

```bash
python3 -c "import os,base64; print(base64.urlsafe_b64encode(os.urandom(32)).decode())"
```

Add to `.env`:
FERNET_KEY=<your-key>
AIRFLOW_UID=50000
MLFLOW_TRACKING_URI=http://mlflow:5000

### 3. Start MLflow (required for training)

```bash
docker compose up mlflow
```

> 👉 Keep this running in a terminal

### 4. Run the DVC pipeline (new terminal)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

dvc repro
```

This will:

- Extract dataset from `data/raw/raw_zipped.zip`
- Run preprocessing
- Train model
- Evaluate model
- Save model to `models/`

### 5. Start full system

```bash
docker compose up --build
```

### 6. Access the system

| Service     | URL                        |
|-------------|----------------------------|
| Frontend    | http://localhost:8501      |
| Backend API | http://localhost:8000/docs |
| MLflow      | http://localhost:5000      |
| Airflow     | http://localhost:8080      |
| Prometheus  | http://localhost:9090      |
| Grafana     | http://localhost:3001      |

## 🧠 How the System Works

### Model Serving Logic

At startup, the backend loads the model in this order:

1. MLflow Production model: `models:/MalariaClassifier/Production`
2. Local trained model: `models/<model>.pth`

> 👉 Since no model is pre-registered, `dvc repro` is required before inference

---

## 🔁 Reproducibility (DVC Pipeline)

The pipeline is defined in `dvc.yaml`.

**Input dataset:**
data/raw/raw_zipped.zip

**Supported formats:**
Parasitized/
Uninfected/

or:
cell_images/
Parasitized/
Uninfected/

### Pipeline Stages

1. `prepare_raw_data` → unzip dataset
2. `eda` → data analysis
3. `preprocess` → splitting
4. `resize` → image normalization
5. `train` → model training
6. `evaluate` → evaluation + metrics

### Run pipeline

```bash
dvc repro
```

### Useful commands

```bash
dvc dag
dvc status
dvc metrics show
dvc plots show
```

## 🧪 Using the Application

### Inference Flow

1. Open Streamlit UI
2. Upload image

Backend will:

- Load model
- Predict label
- Compute confidence + entropy
- Log metrics

> 👉 Low-confidence / OOD samples are stored in feedback queue

---

## 🔁 Retraining (Airflow)

Airflow manages the feedback loop.

New labelled data added to:
data/feedback/labelled/

DAG performs:

- Preprocessing
- Fine-tuning
- Evaluation
- Promotion to MLflow

- `mlruns/` is **NOT** tracked → generated at runtime
