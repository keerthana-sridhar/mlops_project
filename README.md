# 🦠 Malaria Cell Classification MLOps System

End-to-end MLOps pipeline for malaria cell classification with:

- FastAPI backend (inference + APIs)
- Streamlit frontend (user interface)
- DVC (reproducible pipeline)
- MLflow (experiment tracking + model registry)
- Airflow (retraining pipeline)
- Prometheus + Grafana + Alertmanager (monitoring)

---

# 🚀 Quick Start (Evaluator Guide)

## 1. Clone the repository

```bash
git clone <repo-url>
cd <repo>
2. Setup environment
cp .env.example .env

Generate Airflow Fernet key:

python3 -c "import os,base64; print(base64.urlsafe_b64encode(os.urandom(32)).decode())"

Add it to .env:

FERNET_KEY=<your-key>
3. ⚠️ IMPORTANT — Run the pipeline FIRST

This project does NOT ship a trained model.

You must run:

dvc repro

👉 This will:

Extract dataset from data/raw/raw_zipped.zip
Run preprocessing
Train model
Evaluate model
Save model to models/
4. Start the full system
docker compose up --build
5. Access the system
Service	URL
Frontend	http://localhost:8501

Backend API	http://localhost:8000/docs

MLflow	http://localhost:5000

Airflow	http://localhost:8080

Prometheus	http://localhost:9090

Grafana	http://localhost:3001
🧠 How the System Works
Model Serving Logic

At startup, the backend loads model in this order:

MLflow Production model (if exists)

models:/MalariaClassifier/Production

Local trained model:

models/<model>.pth

👉 Since no model is pre-registered:

dvc repro is REQUIRED before inference

🔁 Reproducibility (DVC Pipeline)

The pipeline is defined in dvc.yaml.

Input dataset
data/raw/raw_zipped.zip

Supported formats:

Parasitized/
Uninfected/

or:

cell_images/
  Parasitized/
  Uninfected/
Pipeline Stages
prepare_raw_data → unzip dataset
eda → data analysis
preprocess → splitting
resize → image normalization
train → model training
evaluate → evaluation + metrics
Run pipeline
dvc repro
Useful commands
dvc dag
dvc status
dvc metrics show
dvc plots show
🧪 Using the Application
Inference flow
Open Streamlit UI
Upload image
Backend:
loads model
predicts label
computes confidence + entropy
logs metrics
Low-confidence/OOD images → stored in feedback queue
🔁 Retraining (Airflow)

Airflow handles feedback loop:

New labelled data added to:

data/feedback/labelled/
DAG triggers:
preprocessing
fine-tuning
evaluation
promotion to MLflow
📊 Monitoring
Prometheus → collects metrics
Grafana → dashboards
Alertmanager → sends alerts to backend
🧪 Testing
pytest
📁 Repository Structure
src/                # backend + training code
